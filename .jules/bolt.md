## 2025-02-23 - optimize-mouse-interaction
**Learning:** High-frequency events like `mousemove` trigger expensive re-renders when using `useState`. Using `framer-motion`'s `useMotionValue` and `useTransform` allows updating animations completely outside the React render loop.
**Action:** For interactive animations dependent on pointer events, prefer `useMotionValue` over React state and decompose complex components so only affected children update via `useMotionValueEvent` or direct binding.
## 2025-02-24 - optimize-scroll-listeners\n**Learning:** Implementing custom window scroll listeners with `requestAnimationFrame` inside React `useEffect` hooks is redundant and less efficient when Framer Motion is already deeply integrated in the app. Framer Motion's `useScroll` and `useMotionValueEvent` run on a single optimized projection loop, whereas multiple custom `passive` scroll listeners can cause main-thread contention and layout thrashing if reads like `window.scrollY` aren't perfectly coordinated.\n**Action:** Use `framer-motion`'s `useScroll()` and `useMotionValueEvent(scrollY, "change", ...)` to replace manual `window.addEventListener('scroll')` and `requestAnimationFrame` boilerplate across all components tracking scroll position.
