# Contributing

First off, thanks for taking the time to contribute! Contributions, issues, and feature requests are welcome.

## Development Workflow

1. You fork the repository.
2. You create your feature branch (`git checkout -b feature/AmazingFeature`).
3. You commit your changes (`git commit -m 'Add some AmazingFeature'`).
4. You push to the branch (`git push origin feature/AmazingFeature`).
5. You open a Pull Request.

## Pull Request Guidelines

- You ensure your code follows the existing style guidelines.
- You include appropriate documentation for new features.
- You provide a clear and descriptive PR title and description.
