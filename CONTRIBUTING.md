# Contributing to Portfolio Website

First off, thank you for considering contributing to the Portfolio Website template! It's people like you that make this a great resource.

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue on GitHub with:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment (Node.js version, OS, browser)

### Proposing Enhancements

If you have an idea to make this template better:
1. Open an issue to discuss your proposal before starting work.
2. Provide a clear use case for the enhancement.
3. Once approved, you can submit a Pull Request.

## Development Setup

1. Fork the repository and clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Portofolio-web.git
   cd Portofolio-web
   ```

2. Install dependencies:
   ```bash
   pnpm install
   ```

3. Create a branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. Make your changes and run the development server to test:
   ```bash
   pnpm dev
   ```

5. Before committing, ensure all checks pass:
   ```bash
   pnpm lint
   pnpm test
   pnpm build
   ```

## Pull Request Process

1. Ensure your code follows the existing style and conventions.
2. Update the README.md with details of changes, if applicable.
3. Submit your PR with a clear description of the changes and the problem they solve.
4. Wait for a review from the maintainers.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
