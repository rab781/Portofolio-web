# Contributing to Portfolio Website

First off, thank you for considering contributing to this portfolio website! It's people like you that make this template a great tool for everyone.

## Setting Up Locally

To get the project running locally, follow these steps:

1. **Fork the repository** on GitHub.
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/your-username/Portofolio-web.git
   cd Portofolio-web
   ```
3. **Install dependencies** using `pnpm` (required):
   ```bash
   pnpm install
   ```
4. **Start the development server**:
   ```bash
   pnpm dev
   ```

## Development Guidelines

- **Code Style**: We use ESLint and Prettier. Run `pnpm lint` to check for any linting issues.
- **Testing**: Run tests using `pnpm test`. All new features should be accompanied by tests where applicable.
- **Building**: Make sure the project builds successfully by running `pnpm build` before submitting your changes.

## Pull Request Process

1. **Create a branch** for your feature or bugfix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Commit your changes**. Write clear, concise commit messages.
3. **Push to your fork** and submit a **Pull Request** to the `main` branch of the original repository.
4. **Review Process**: An maintainer will review your PR. Be prepared to address any feedback or make changes if requested.

## Reporting Issues

If you find a bug or have a feature request, please [open an issue](https://github.com/rab781/Portofolio-web/issues) on GitHub. Please include as much detail as possible to help us understand and resolve the issue.

Thank you for contributing!
